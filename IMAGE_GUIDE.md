## 🎯 Image Specifications

**General Requirements:**
- **Format:** JPG, PNG, or WebP
- **Quality:** High resolution (minimum 1920px width for hero images, 800px for product images)
- **File Size:** Optimize for web (compress images to under 500KB where possible)
- **Aspect Ratios:** Maintain consistent aspect ratios within each section

## 🔧 How to Replace Images

### Step 1: Organize Your Images
Place your images in the appropriate folders under `src/assets/` following the structure above.

### Step 2: Import Images in Components
Update the component files to import actual images instead of placeholders.

**Example for Hero Carousel:**
```javascript
// At the top of HeroCarousel.jsx
import textileHero from '../assets/hero/textile-division-hero.jpg';
import fmcgHero from '../assets/hero/fmcg-division-hero.jpg';
// ... etc

// Then in the slides array:
const slides = [
  {
    id: 1,
    image: textileHero, // Instead of placeholder URL
    // ... rest of slide data
  },
  // ... more slides
];
```

### Step 3: Update Image URLs
Replace all placeholder URLs (e.g., `https://via.placeholder.com/...`) with imported image references.

## 📋 Image Optimization Tips

1. **Compress Images:** Use tools like TinyPNG, ImageOptim, or Squoosh
2. **Use WebP Format:** For better compression and quality
3. **Lazy Loading:** Large images are already set to lazy load
4. **Alt Text:** Ensure all images have descriptive alt text for accessibility
5. **Responsive Images:** Consider using `srcset` for different screen sizes
